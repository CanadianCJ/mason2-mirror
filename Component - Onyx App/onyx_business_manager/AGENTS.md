# Onyx Business Manager Scope

## Focus
1. Business manager UX for core workflows.
2. Core flows: dashboard, tasks, invoices, and CRM basics.
3. Performance and responsiveness.
4. Security and data handling safety.

## Allowed Work
1. Flow fixes and UX refinements that reduce friction in core tasks.
2. Measurable performance improvements in Onyx business manager screens/flows.
3. Security hardening tied to Onyx business manager behavior.

## Not Allowed
1. Unrelated features (for example games or novelty modules).
2. Generic redesigns without measurable UX wins.
3. Broad visual churn that does not improve core flow outcomes.

## Required For Every Onyx Change
1. "Why this helps Onyx users" statement tied to a core flow.
2. Minimal UI changes only; avoid unnecessary visual rewrites.
3. Deterministic validation steps for affected dashboard/tasks/invoices/CRM flow.
